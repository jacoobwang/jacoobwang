Linux定位问题常用的命令

发表于 2018-03-26

当你在linux上撸的时间越多，你就会linux的命令越发熟悉。这篇文章主要记录我觉得比较实用的一些命令！

top

查看服务器资源实用情况！类似于windows下的资源管理器！但比资源管理器更清晰明了，可以清楚看到cpu内存等实用情况。

上面是我在某台机器上执行top时的部分输出。简单介绍下：

  1. 20:03:23 当前系统时间
  2. up 293 days, 2:31 运行的时长
  3. 1 user 当前有1个用户登录
  4. load average: 0.74, 0.49, 0.47 负载情况:1分钟 5分钟 15分钟
  5. Tasks 系统有149个进程 1个正在运行 148个在休眠 0个stopped 0个zombie（僵尸进程）

  6. Cpu 系统cpu使用情况

    | 指标 | 释义 |
    | ——| —— |
    | us | 用户空间占用cpu比例 |
    | sy | 内核空间占用cpu比例 |
    | ni | 改变过优先级的进程占用cpu比例 |
    | id | 空闲cpu比例 |
    | wa | IO等待占用cpu比例 |
    | hi | 硬中断占用cpu |
    | si | 软中断占用cpu |

  7. Mem 系统内存使用情况：total 物理内存总量；used 使用中的内存；free 未使用内存总量;buffers 缓存的内存总量。
  8. Swap 交换分区信息：total 交互去总量；used 使用的交换区；free 未使用的交换区；cached 缓存的交换区总量。
  9. 接下来就是各个进程的一些情况，比较简单，这里就不说了。

top 命令小技巧

  • 1(数字1)：列出所有的逻辑CPU负载情况
  • z: 高亮显示
  • P: 按cpuz占用高低排序
  • M: 按内存占用高低排序
  • shift < : 向下翻页
  • shift > : 向上翻页
  • c: 显示进程命令的全路径与参数 （这个非常有用，帮助我们找到一些僵尸进程或者cpu过高的进程信息）
  • H: H:显示线程，默认只显示进程

netstat

  1. netstat -tnl 列出所有正在被监听的tcp端口。任何网络服务的后台进程都会打开一个端口，用于监听接入的请求。这些正在监听的套接字也和连接的套接字一样，也能被 netstat 列出来。

  2. netstat -unl 列出所有正在被监听的udp端口。

  3. netstat -an|grep xxxx 查看xxx端口是否被占用。

  4. netstat -p 显示进程名称和pid。

  5. netstat -ant |grep “:80”|awk ‘{print $5}’|awk -F: ‘{print $1}’|sort|uniq -c|sort -nr|head -20 查看连接某服务端口最多的的IP地址

  6. netstat -ant|grep -i “TIME_WAIT”|wc -l 查看Tcp状态TIME_WAIT的数量

  7. netstat -ant | grep -e 127.0.0.1 -e 0.0.0.0 -e ::: -v | awk ‘/^tcp/ {++state[$NF]} END {for(i in state) print i,”\t”,state[i]}’ TCP各种状态列表

tcpdump

它可以打印所有经过网络接口的数据包的头信息，也可以使用-w选项将数据包保存到文件中，方便以后分析。

  1. tcpdump -i eth1 查看eth1网卡的数据包，如果不指定网卡，默认监视第一个网络接口，一般是eth0。

  2. tcpdump host 10.184.89.19 截获所有10.184.89.19 的主机收到的和发出的所有的数据包。

  3. tcpdump -i eth1 -X -s 0 -c 10 host 10.184.89.19 and tcp port 3800 -w /tmp/tcp.cap 将3800端口上收到的数据包写入到tcp.cap文件中。

    • s 0 如果不设置默认将会是68字节，需要注意的是, 采用长的抓取长度(nt: snaplen比较大), 会增加包的处理时间, 并且会减少tcpdump 可缓存的数据包的数量， 从而会导致数据包的丢失. 所以, 在能抓取我们想要的包的前提下, 抓取长度越小越好.把snaplen 设置为0 意味着让tcpdump自动选择合适的长度来抓取数据包。
    • c 10 指定包的个数，这里是抓取10个数据包后自动退出监视
    • X 当分析和打印时, tcpdump 会打印每个包的头部数据, 同时会以16进制和ASCII码形式打印出每个包的数据
    • w 写入的文件
    • host 10.184.89.19 and tcp port 3800 指定监听的主机和端口
