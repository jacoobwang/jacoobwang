Mysql PDO执行报错： Cannot execute queries while other unbuffered queries are active

发表于 2018-01-16

我负责的一个问卷系统是用php开源系统limesurvey搭建的，limesurvey是基于yii framework 1.0搭建。最近上线一套新的模板，在测试环境添加问卷逻辑时发现pdo执行报错：

报错内容

当存在一个未缓冲query且该连接还是active状态下时，pdo不会执行新的query。可以考虑使用fetchAll来执行新查询！另外，如果你使用的是mysql，你可以打开PDO关于MYSQL_ATTR_USE_BUFFERED_QUERY属性的设置！

通常是由 fetch 而不是 fetchAll 引起的。 但是在这种情况下，只有一个结果，它正在被获取。 同样，如果只执行第一次 prepared 语句，错误就不会发生。 只有当第一个语句被 prepared 两次时才会发生。 它也只有当 PDO::ATTR_EMULATE_PREPARES 是 false 时才会发生。

解决方法

  1. 使用缓冲查询！

  2. 使用closeCursor，在上一个查询中释放连接！PDOStatement::closeCursor() 释放与服务器的连接，以便后面的连接可以使用。

  3. 如果使用了fetch，则替换成fetchAll

    可以参考下php官网中的一段示例代码。http://php.net/manual/zh/pdo.exec.php#75070

以上三种方法是基于googole找到的一些解决方法！
