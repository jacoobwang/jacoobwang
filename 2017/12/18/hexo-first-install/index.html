终于决定还是搞一个hexo静态博客

发表于 2017-12-18

好几年前我有一个用php撸的博客，用了一段时间后，转用wordpress。毕竟当时wordpress太火爆，一度靠wordpress还赚了一点外快！时钟转到2017年，wordpress貌似不再流行，php也不再流行。在网络上瞎逛，看到很多人用hexo，遂决定用hexo来建一个博客！

放弃wordpress还有另外一个原因，php站点是需要服务器资源的，就算是最便宜的搬瓦工一年也要耗费几十块大洋来支付费用，屌丝啊，总是不想付费。刚好看到github pages不用钱，而github pages正好支持部署静态站点，心中略喜，finally决定就是hexo拉。

部署到Github Pages

如果你还不太了解什么是github pages的话，可以看下这个页面https://pages.github.com/。简单说他就是github推出的一个静态站点服务！

Github Pages的规则

首先得知道Github pages的规则：每个Github账号（比如Username）下面只能建立一个Pages，且命名必须符合这样的规则：”username/username.github.io”

创建成功后，username.github.io就是你的域名（当然你可以通过别名解析绑定自己的域名）。
